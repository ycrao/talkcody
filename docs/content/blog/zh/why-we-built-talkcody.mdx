## 我们看到的问题

作为开发者,我们都曾体验过在 IDE、文档、Stack Overflow 和各种 AI 聊天界面之间频繁切换的挫败感。虽然 AI 编程助手已经变得非常强大,但使用它们的体验往往感觉支离破碎,与我们实际的开发工作流程脱节。

## 我们的愿景

我们想要创造一些不同的东西——一个原生桌面应用程序,将 AI 辅助直接带入您的开发环境,同时保持 macOS 用户期望的速度、可靠性和原生体验。

### 设计原则

在构建 TalkCody 时,我们专注于三个核心原则:

#### 1. 原生优先

我们选择 Tauri 2 而非 Electron,因为我们相信桌面应用程序应该具有原生体验。TalkCody 启动迅速,占用内存极少,并与 macOS 的暗色模式、通知和系统偏好设置等功能无缝集成。

#### 2. 现代技术

通过使用 React 19、TypeScript 和 Vercel AI SDK,我们确保 TalkCody 建立在坚实的基础之上,能够随着快速变化的 AI 领域不断演进。我们可以轻松集成新的模型和功能。

#### 3. 开发者体验

TalkCody 的每个方面都以开发者为中心进行设计。从键盘快捷键到语法高亮,我们专注于创造一种自然的体验,提升您的生产力。

## 技术栈

### 前端: React 19 + TypeScript

我们选择 React 19 是因为它出色的开发者体验和强大的生态系统。TypeScript 确保类型安全,并使代码库在添加新功能时易于维护。

### 后端: Tauri 2 + Rust

Tauri 在 Web 技术和原生性能之间提供了完美的平衡。Rust 的安全保证让我们对 TalkCody 的安全性和可靠性充满信心。

### AI: Vercel AI SDK 5.0

Vercel AI SDK 允许我们集成多个 AI 提供商,并轻松在模型之间切换。这种灵活性意味着 TalkCody 可以使用当前和未来最好的模型。

### 数据库: SQLite

对于本地数据存储,SQLite 提供了轻量级、可靠的解决方案,不需要任何外部服务或配置。

## TalkCody 的与众不同之处

### 有意义的对话

TalkCody 在整个对话过程中保持上下文,理解您的代码和需求的演变。它不仅仅是回答单个问题——而是成为真正的结对编程伙伴。

### 隐私优先

您的所有对话和代码都保存在本地机器上。我们不会将您的数据发送到外部服务器(除了必要的 AI 模型 API 调用,这是服务正常工作所需的)。

### 可扩展架构

TalkCody 的设计具有可扩展性。我们正在开发一个插件系统,允许社区添加新功能、集成和 AI 提供商。

## 未来之路

这仅仅是开始。以下是我们正在开发的功能:

- **插件系统** - 允许开发者使用自定义功能扩展 TalkCody
- **更多 AI 提供商** - 支持本地模型和更多云端提供商
- **团队功能** - 与团队分享对话和最佳实践
- **IDE 集成** - 与流行的 IDE 和编辑器深度集成
